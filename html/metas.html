Java EE

Sin dudas, uno de los conceptos que más me llamó la atención una vez que empezé a desarrollar aplicaciones elaboradas en Java, que requerían de distintos patrones de diseño y un uso extensivo de su API es el conjunto de tecnologías Java EE para la producción de aplicaciones empresariales.
Java EE consta de una serie de APIs, especificaciones y tecnologías apuntadas al desarrollo de aplicaciones grandes, mantenibles, extensibles y escalables, digamos que sería uno de los puntos más altos del desarrollo en Java. Si bien ya experimenté un poco con tecnologías relacionadas (Servlets, JDBC entre otros), una de mis mayores ambiciones para este año es comenzar en el estudio a fondo de todo el paquete de Java EE.

Desarrollo de aplicaciones nativas en Android

Desde que empecé a estudiar Java, una de las preguntas que empecé a hacerme a mí mismo era en qué áreas se aplicaba, de ahí nació mi interés por el desarrollo de aplicaciones en Android, por lo que uno de mis objetivos más inmediatos es encontrar algún curso sobre el cual no sólo aprenda las bases, si no que a lo largo de él se requiera el desarrollo de aplicaciones para ganar experiencia práctica. Sin embargo, un punto sobre el cual tengo que decidir todavías es si apuntar al desarrollo en Android con Java o con Kotlin, lenguaje del que todavía no sé mucho pero que no tendría problemas en aprender si me permite desarrollar de forma más fácil y/ó eficiente.

Certificaciones OCA y OCP para Java SE 8 provistas por Oracle

Otra meta fuerte que espero poder lograr es la certificación de mis aptitudes en Java a través de los exámenes Java SE 8 Programmer I y Java SE 8 Programmer II (también conocidos como OCA) los cuales en su conjunto garantizan el título de OCP para Java SE 8. Ya cuento en mi posesión con muy buen material de estudio, además de haberme informado sobre la modalidad de los exámenes y donde tomarlos. La decisión de certificarme en la SE 8 en vez de la SE 11 (la cual es la versión más nueva con soporte a largo plazo) es basada en el concepto de modularidad que esta última introduce, concepto en el que pienso adentrarme sólo una vez que tenga un muy buen manejo de Java y del desarrollo de software en general.

JavaScript

JavaScript es un lenguaje nuevo para mí. Si bien hasta hace poco pensaba que era sólo un lenguaje usado para el desarrollo de front end en sitios web, hasta qué comencé a investigar sobre sus diversas áreas de aplicación que tiene, y por lo visto tiene un futuro muy prometedor. Si bien tengo una experiencia más que nada anecdótica con JS (habiéndo usado un poco en algún que otro proyecto), lo encuentro un lenguaje muy amigable y que en el que me veo desarrollando software en distintas áreas no sólo dentro del mundo web.
